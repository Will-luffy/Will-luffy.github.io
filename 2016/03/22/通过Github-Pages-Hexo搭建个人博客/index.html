注册Github Pages

github pages是一套静态网站搭建方案，通过github进行代码和服务的托管。github pages主要分两种，user/organization pages和project pages，前者主要用于搭建个人或组织网站，后者主要对应github上的每个repository。

  1. 首先需要有github帐号，如果没有，请自行申请
  2. 创建repository，名称一定要为{username}.github.io，其中{username}为你的github账户名
  3. 在代码库（repository）中增加index.html，然后访问http://{username}.github.io就能看到index.html的展示了

通过Automatic Pages Generator生成页面

  • 如果是要采用hexo搭建，建议直接跳过该步骤，hexo代码会将这个步骤的代码覆盖
  1. 进入代码库的setting
  2. 点击“Launch automatic pages generator”
  3. 编辑Page name、Tagline和Body，Body支持Markdown
  4. 然后点击“Continue to layouts”选择模板

安装配置Hexo

Hexo是一款基于Node.js的静态博客框架，简单、快速。

安装

  • 首先本地需要安装Node.js、Git
  • 安装hexo

    npm install -g hexo

配置

  • 初始化：新建文件夹hexo（这里我刚开始一直以为是需要将github的代码clone下来，其实不然，这个文件夹只是存放hexo依赖模块、配置文件和源码的地方）

    cd hexo
    hexo init

  • 预览

    hexo server
    然后浏览器打开http://localhost:4000，就能看到生成的博客

  • 生成静态页面

    hexo generate

  • 部署：hexo支持多种部署方式，Github、Heroku、Openshift等

    • 部署到github

      hexo deploy

  • 更换主题

    • 到Hexo官网找到喜爱的主题，获取其github地址
    • 在hexo/themes目录下，执行git clone ‘主题github地址’
    • 然后修改hexo/_config.yml, 将theme设置为下载的主题，还可以将language设置为需要的语言（一般主题都支持多种语音）
